1. Executive Summary

The application is a frontend-first Lodge Management Software built in React and Vite. It is designed to help one lodge manage daily operations such as room status, guest intake, employee records, service menus, expense tracking, inventory, housekeeping, billing, and admin access.

Business Goal

Launch quickly for a single property, demo easily to customers, and keep the software affordable to host and support.

Current Delivery Model

Static frontend hosting plus optional Supabase cloud storage, with local browser storage fallback when cloud sync is not configured.

5. Hosting, Database And Deployment

Area Recommended Option Reason Operational Note
Frontend Hosting Cloudflare Pages or Netlify Good fit for static Vite builds Easy GitHub-based deployment
Database Supabase Fastest cloud setup for a one-lodge app Current code is already prepared for it
Current Data Model app_state JSON row Simple and low-friction for MVP use Best for one lodge and early trials
Future Upgrade Separate normalized tables Better reporting, security, and growth Recommended before scaling to many customers

Deployment Checklist

  • Push code to GitHub
  • Create Supabase project
  • Run supabase-schema.sql
  • Create environment variables
  • Connect repo to hosting platform
  • Build with npm run build
  • Publish the dist folder
